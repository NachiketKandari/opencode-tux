## Key Design Decisions

1. **No SQLAlchemy** — Oracle's long-term driver strategy favors `oracledb` directly. Connection pooling via `oracledb.create_pool_async()`.
2. **SQL preserved verbatim** — every `EXEC SQL` from `.pc` files appears as a raw Python string, no ORM translation.
3. **4 runtime dependencies** — `oracledb`, `structlog`, `prometheus_client`, `typer`.
4. **Async throughout** — `oracledb` async pool enables concurrent I/O at millions-of-rows scale.
5. **BatchProcess base class** — lifecycle (`init_db` → `run` → `done`) mirrors Tuxedo's `tpsvrinit` → service → `tpsvrdone` pattern.
